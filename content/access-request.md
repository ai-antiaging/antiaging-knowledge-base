# 📧 访问权限申请

> **本网站为私有知识库** · 仅对授权用户开放

---

## 🔐 为什么需要申请？

本网站包含：
- 📊 未发表的研究报告和数据
- 💊 临床试验内部分析
- 🔬 前沿研究解读和趋势分析
- 🧰 专业研究工具和数据集

为确保内容质量和合规性，我们实行邀请制访问。

---

## 📝 申请条件

### 符合以下任一条件即可申请：

1. **研究人员**
   - 高校或研究机构在职研究人员
   - 研究方向：衰老生物学、转化医学、生物信息学等
   - 需提供机构邮箱或研究主页

2. **临床医生**
   - 执业医师或临床研究人员
   - 关注抗衰老医学应用
   - 需提供执业医师资格证编号

3. **投资者**
   - 生物科技领域投资人
   - 关注抗衰老产业趋势
   - 需提供机构证明

4. **其他专业人士**
   - 科学记者、科普作家
   - 政策制定者、监管机构人员
   - 需说明使用目的

## 📮 申请流程

### 方式 1：邮件申请（推荐）

1. **发送邮件至**：`your-email@example.com`

2. **邮件主题**：网站访问申请 - [你的姓名] - [你的机构]

3. **邮件内容**：
   ```
   姓名：[你的全名]
   机构：[你的工作单位]
   职位：[你的职位]
   邮箱：[你的工作邮箱]
   研究方向：[你的研究领域]
   
   申请理由：
   [请说明你为什么需要访问本网站，计划如何使用本站内容]
   
   承诺：
   我承诺本网站内容仅用于个人学习和研究，不会未经授权的传播或用于商业用途。
   
   签名：[你的姓名]
   日期：[申请日期]
   ```

4. **等待审核**：通常在 3-5 个工作日内回复

5. **获取密码**：审核通过后，你会收到包含访问密码的邮件

---

### 方式 2：GitHub Issue

1. **访问 GitHub 仓库**：
   ```
   https://github.com/muwangdao/antiaging-knowledge-base
   ```

2. **创建 Issue**：
   - 点击 "Issues" → "New Issue"
   - 选择 "Access Request" 模板
   - 填写申请信息

3. **等待审核**：管理员会在 Issue 中回复

---

### 方式 3：推荐人制度

如果你认识已经获得访问权限的用户：

1. **获取推荐**：请现有用户为你推荐
2. **简化流程**：推荐申请可加快审核速度
3. **直接获取**：推荐人可以直接分享临时访问码

## 🔑 访问权限说明

### 权限等级

| 等级 | 权限 | 适用人群 |
|------|------|---------|
| **基础访问** | 查看公开内容 | 所有授权用户 |
| **研究访问** | 查看研究数据 + 下载工具 | 研究人员、临床医生 |
| **专业访问** | 全部权限 + API 访问 | 合作机构、投资者 |

### 密码有效期

- **临时密码**：7 天（用于试用）
- **标准密码**：90 天（标准授权）
- **长期密码**：1 年（合作机构）

密码到期前会收到续期提醒。

## ⚠️ 使用条款

### 允许的行为

- ✅ 个人学习和研究
- ✅ 学术引用（需注明出处）
- ✅ 内部团队分享（需团队成员也获得授权）
- ✅ 打印用于个人参考

### 禁止的行为

- ❌ 公开传播网站内容
- ❌ 商业用途（需另行授权）
- ❌ 批量下载或爬取数据
- ❌ 分享访问密码给未授权人员
- ❌ 反向工程或破解网站

### 违规后果

违反使用条款可能导致：
1. 访问权限立即终止
2. 永久禁止再次申请
3. 严重情况可能承担法律责任

## ❓ 常见问题

**Q: 申请被拒绝后可以再次申请吗？**
A: 可以，建议等待 30 天后重新申请，并补充更多相关信息。

**Q: 密码忘记了怎么办？**
A: 发送邮件至 `your-email@example.com`，说明情况，我们会重新发送密码。

**Q: 可以分享我的密码给同事吗？**
A: 不可以。每位用户需要单独申请。但可以推荐同事申请。

**Q: 密码多久更换一次？**
A: 标准密码有效期 90 天，到期前会提醒续期。建议定期更换密码。

**Q: 机构多人使用怎么办？**
A: 请申请机构授权，可以获取机构专属访问方案。
